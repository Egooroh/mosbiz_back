# Mosbiz REST API Backend

Серверная часть приложения, разработанная на базе фреймворка Laravel 10. Проект предоставляет RESTful API для клиентской части и включает полноценную панель администрирования для управления контентом, построенную на базе Filament v3.

## Системные требования

* PHP 8.1 или выше
* Composer 2.x
* MySQL 8.0+
* Локальный веб-сервер (OpenServer) или настроенный хостинг

## Развертывание проекта

1. **Клонирование репозитория и установка зависимостей**
   ```bash
   git clone <url-вашего-репозитория>
   cd mosbiz-api
   composer install
   ```

2. **Настройка окружения**
   ```bash
   cp .env.example .env
   ```
   Укажите реквизиты доступа к БД в файле `.env`:
   ```env
   DB_CONNECTION=mysql
   DB_HOST=127.0.0.1
   DB_PORT=3306
   DB_DATABASE=имя_вашей_базы
   DB_USERNAME=пользователь_базы
   DB_PASSWORD=пароль
   ```

3. **Инициализация приложения**
   ```bash
   php artisan key:generate
   php artisan migrate
   php artisan storage:link
   ```

4. **Создание администратора и запуск**
   ```bash
   php artisan make:filament-user
   php artisan serve
   ```

## Панель администрирования

Управление всем контентом осуществляется через встроенную панель, реализованную с помощью Filament v3 (TALL-стек). 

* **URL для входа:** `http://localhost:8000/admin` (или `ваш-домен/admin` на сервере).
* **Аутентификация:** Используются учетные данные, созданные на этапе развертывания (`php artisan make:filament-user`).

### Основные разделы панели:
* **Настройки сайта:** Единая точка управления первым экраном приложения. Позволяет загружать логотип, фоновое изображение баннера, редактировать тексты, а также динамически добавлять/удалять иконки социальных сетей и карточки статистики (реализовано через JSON-массивы).
* **Сотрудники:** Управление составом команды. Включает загрузку фотографий, выбор направления (через жестко заданный Select) и логический переключатель (тумблер) для выделения руководителя.
* **Мероприятия:** Управление карточками событий, установка дат проведения, категорий и описаний.
* **Заявки:** Раздел для обработки входящих лидов с сайта. Создание новых записей через админку заблокировано — заявки поступают строго через API.

## Документация API

Проект использует пакет Scribe для генерации интерактивной документации. Для её обновления выполните команду:
```bash
php artisan scribe:generate
```
Документация доступна по адресу: `/docs`.

## Описание маршрутов API

Все запросы и ответы осуществляются в формате `application/json`. Базовый префикс маршрутов: `/api/v1`.

### Настройки сайта
* **GET /api/v1/settings**
  Возвращает глобальные конфигурационные данные для первого экрана.
  * **Успешный ответ (200):** Объект `data`, содержащий:
    * `header_logo` (string|null) — абсолютный URL логотипа.
    * `header_button_text` (string) — текст главной кнопки.
    * `header_phone` (string) — контактный телефон.
    * `social_links` (array) — массив объектов `{"icon": "url", "url": "url"}`.
    * `hero_image` (string|null) — абсолютный URL фона.
    * `hero_title` (string) — заголовок баннера.
    * `hero_subtitle` (string) — подзаголовок баннера.
    * `statistics` (array) — массив объектов `{"value": "строка", "description": "строка"}`.
  * **Ошибка (404):** Возвращается, если запись настроек еще не инициализирована в базе данных.

### Мероприятия
* **GET /api/v1/events**
  Возвращает расписание мероприятий, отсортированное по дате начала по возрастанию.
  * **Успешный ответ (200):** Массив объектов в ключе `data`. Каждый объект содержит: `id`, `title`, `start_date`, `end_date`, `category`, `description`, `image_url` (абсолютный URL или null).

### Команда (Департамент)
* **GET /api/v1/team**
  Возвращает список сотрудников, предварительно отформатированный для фронтенда.
  * **Успешный ответ (200):** Объект `data` с двумя ключами:
    * `head` (object|null) — объект сотрудника, отмеченного как руководитель (`is_head = true`). Включает: `id`, `name`, `position`, `description`, `photo_url`.
    * `members` (array) — массив объектов остальных сотрудников (`is_head = false`).

### Обратная связь
* **POST /api/v1/feedback**
  Принимает данные контактной формы от клиента.
  * **Тело запроса:**
    * `name` (string, required, max 255) — Имя пользователя.
    * `phone` (string, required, max 25) — Контактный номер телефона.
    * `email` (string, required, email, max 255) — Адрес электронной почты.
    * `inn` (string, optional, max 12) — ИНН организации.
  * **Успешный ответ (201):** Объект с флагом `success: true`, сообщением об успехе и `id` созданной заявки.
  * **Ошибка валидации (422):** Объект с флагом `success: false` и массивом `errors` с описанием некорректных полей.
